# Constitution

This repository is the public home of the Lean Context Thesis. It follows the Lean Context standard.

## Hard Stops

- Do not modify published thesis content without author review.
- Do not push directly to main without verifying the build succeeds locally.
- Do not change the design system (colors, typography, layout structure) without explicit approval.
- Do not add dependencies or JavaScript frameworks. This is a static Hugo site. Keep it static.
- Do not remove or rename published URL paths. Existing links must not break.

## Context System

- CONSTITUTION.md (this file): Always loaded. Hard stops only.
- MANIFEST.md: Routing table. Consult before starting any task.
- .context/: Supply documents. Pull on demand when relevant.

## Andon Rule

When you lack sufficient context to proceed with confidence, stop.
Do not guess. Do not fill gaps with assumptions. State what you
believe you are missing and ask. The act of stopping is the system
working, not failing.

When a gap is found, help write the supply document to fill it.
When the problem is solved or the gap is filled, the session ends.
