# The Lean Context Thesis

**How Humans and AI Agents Make Each Other Smarter**

A philosophy, an operation, and a structure for human-agent collaboration.

## What This Is

The Lean Context Thesis argues that untargeted upfront context dilutes AI agents — and that the fix is not better prompts, but better systems. It proposes a three-tier architecture (constitution, manifest, supply) implemented through the spine pattern, maintained by context compaction, and governed by a cultural contract between the human and the agent.

## Key Concepts

- **Constitution** — always loaded, hard stops only, 30–50 lines
- **Manifest** — routing table mapping tasks to supply documents, 10–15 lines
- **Supply** — pulled on demand, one topic per document, invisible knowledge only
- **Andon Rule** — stop and surface uncertainty instead of guessing
- **Session Discipline** — one task, one session, clean workbench
- **Knowledge Boundary** — inferable knowledge stays out, invisible knowledge goes in
