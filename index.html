The Lean Context Thesis

How Humans and AI Agents Make Each Other Smarter

Untargeted upfront context makes AI agents worse, not better. The fix is not better prompts — it is a better system.

End the Context Junk Drawer

The Problem

Stop fighting AI hallucinations by blindly adding more rules. Stacking instructions into a massive, monolithic prompt dilutes the agent's focus and makes failures impossible to debug. We replace fear-based prompt bloat with disciplined, curated context management.

Lean Architecture for AI

The Framework

Treat the context window like a physical workbench. A Constitution for safety. A Manifest for routing. A Supply for depth. Agents explicitly pull only the knowledge they need, exactly when they need it. No cognitive clutter. No blind guessing.

The Amplification Loop

The Human Impact

This is not just about saving tokens — it is about human governance. Agents explicitly pull context or stop when uncertain. Their reasoning becomes visible and traceable. Fear is replaced by discipline. Trust scales. Humans and agents make each other smarter.

Key Concepts

Constitution

Always loaded. Hard stops only. 30–50 lines. If the agent misses it once, the damage is irreversible.

Manifest

A routing table. Maps task patterns to supply documents. The agent sees what is available without holding it.

Supply

Pulled on demand. One topic per document. Invisible knowledge only — what the agent could never infer from code.

Andon Rule

Stop and surface uncertainty instead of guessing. The agent helps fill the gap. The session ends. The system improves.

Session Discipline

One task, one session, clean workbench. When the problem is solved or the gap is filled, the session ends.

Context Compaction

The human maintains the system between sessions so the agent never needs to compress within one.
