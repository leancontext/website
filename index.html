lean context

More instructions make your agents worse.

The Lean Context Thesis is a structural approach to human-agent collaboration. Stop stuffing the context window. Start curating it.

the problem

End the junk drawer

Every mistake triggers more instructions. More instructions dilute focus. A different mistake happens. The loop repeats.

This is the degradation loop — the default trajectory of every project that manages agent context through accumulation rather than curation. Adding rules feels like safety. It is the opposite.

the framework

Lean architecture for AI

Treat the context window like a factory workbench. Only what the current operation needs. Everything else is inventory waste.

A constitution for hard stops. A manifest to route context. A supply pulled on demand. The agent shows its work. The human can trace every decision.

the human impact

The amplification loop

When the agent stops instead of guessing, the gap gets filled. The system improves. Trust builds. The human delegates more. Both sides get smarter.

Fear is replaced by discipline. Sessions end clean. Context lives in the repo — not the conversation. The system remembers so the agent doesn't have to.

how it works

The spine pattern

Every repository gets the same structural skeleton. Three tiers. One contract.

Always loaded

Constitution

Hard stops only. If the agent misses it once, the damage is irreversible. 30–50 lines. Every line load-bearing. The constitution is not a journal — it is a checklist for operating on production.

Always visible

Manifest

A routing table that maps tasks to supply documents. The agent sees what context exists without holding any of it. Shadow board for the workbench. 10–15 lines.

Pulled on demand

Supply

One topic per document. Only invisible knowledge — the things the agent could never discover from code alone. Named by retrieval key. Loaded when earned.

principles

Andon rule

Stop when uncertain. Surface what is missing. Help fill the gap. The system gets smarter every time the agent stops instead of guessing.

Session discipline

One task. One session. Clean workbench. Problem solved or gap filled — the session ends. Context lives in the repo, not the conversation history.

Knowledge boundary

Inferable knowledge stays out. The agent can read the code. Invisible knowledge goes in — the decisions, the failures, the constraints no code can encode.

Context compaction

The human maintains the system between sessions. Demote, prune, validate. Sacred is the enemy of lean. If the agent never needs runtime compression, the system is working.
